## 核心问题

面对长文档、多来源材料和复杂上下文时，信息该怎样组织。

## 这节学什么

学习上下文不是堆材料，而是选择、排序、切块和限定。

## 为什么这节重要

很多初学者以为“大模型支持长上下文”就等于“把全文丢进去就行”。实际情况恰恰相反。长文本任务里，最常见的问题不是模型完全看不见，而是：

- 关键信息被淹没
- 中间部分容易被忽略
- 指令埋得太深
- 材料太多导致幻觉和跑题

所以长文本处理的关键，不是“输入更多”，而是“组织得更聪明”。

## 关键概念

- chunking
- windowing
- context ordering
- evidence selection

## 通俗解释

### 1. 为什么不能把全文直接丢给模型

原因主要有四个：

#### 第一，注意力不是均匀分布的

模型通常对开头和结尾更敏感，中间部分更容易被忽视。这就是长文本处理中常见的“中间遗失”问题。

#### 第二，指令会被材料稀释

如果你把关键要求写在最前面，然后后面贴十几页内容，模型在生成回答时，未必还能牢牢记住最开始那句指令。

#### 第三，成本和速度都会增加

上下文越长，处理越慢，成本越高，而且不代表质量同步变好。

#### 第四，幻觉风险会上升

材料太多、太杂时，模型更容易把不相关的信息混在一起，强行编出看似合理的解释。

### 2. 什么是上下文组织

上下文组织就是在把内容交给模型之前，先做一层工程化处理：

- 只保留必要内容
- 把长文切块
- 调整材料顺序
- 压缩冗余信息
- 只把最相关的证据送给模型

这不是“少给信息”，而是“给更有用的信息”。

## 五个核心概念展开

### 1. Chunking

chunking 就是把长文本切成更小的块。

适合场景：

- 长文摘要
- 多文档问答
- 文本比较
- RAG 前处理

常见方式：

- 按段落切
- 按主题切
- 按固定长度切

### 2. Windowing

windowing 是在分块时让相邻块保留一部分重叠内容。

例如：

- 每块 1000 字
- 相邻块重叠 200 字

作用是避免关键句子刚好被切断，导致上下文断裂。

### 3. Context Ordering

context ordering 就是安排信息顺序。一个实用原则是：

- 背景材料放前面
- 最关键的任务要求放最后

因为模型在生成回答时，越靠近末尾的内容通常越容易被优先记住。

### 4. Summary Compression

summary compression 就是先压缩，再继续处理。

常见做法：

1. 先对每个块生成局部摘要
2. 再把这些摘要汇总
3. 最后生成全局总结

这特别适合：

- 长报告
- 多轮对话
- 书籍章节
- 长会议纪要

### 5. Evidence Selection

evidence selection 的核心是：不要让模型在大海里捞针，而是先把针附近那杯水取出来。

这就是检索增强思路的核心：

- 先找相关证据
- 再基于证据回答

它特别适合：

- 知识库问答
- 合同问答
- 代码定位
- 长文档事实查找

## 不同任务的组织策略

### 1. 摘要任务

适合：

- 分块 + 局部摘要 + 全局综合

### 2. 比较任务

适合：

- 先提取两边对应部分
- 再对比差异

不要直接把两份百页文件一起扔进去让模型“自己看”。

### 3. 精准问答

适合：

- 检索相关块
- 限定模型只基于这些块回答

### 4. 写作支持

适合：

- 背景材料前置
- 写作要求后置
- 用标签清楚分隔资料

### 5. 研究分析

适合：

- 多文档分隔
- 任务拆解
- 先提取，再比较，再综合

## 10个可直接上手的例子

### 例子1：长篇会议纪要摘要

先把会议纪要分成若干段，再分别提炼每段要点，最后合成总决议。

### 例子2：专著内容提炼

按章节切块，每章先总结，再让模型写全书概览。

### 例子3：用户反馈批量处理

把 10000 条反馈按批次切分，每批分类后再汇总统计。

### 例子4：长合同排雷

用滑动窗口切合同，防止“违约条款”恰好被切断。

### 例子5：多行业报告整合

给不同报告加清晰标签，再要求模型找交叉机会。

### 例子6：大型代码库审查

先用搜索或日志定位相关函数，再只把这些代码块交给模型分析。

### 例子7：客服知识库问答

先检索说明书里最相关的段落，再要求模型仅基于这些段落作答。

### 例子8：财报对比

先从两份年报中提取营收、利润、研发支出，再做表格对比。

### 例子9：长篇小说设定一致性

把世界观设定和人物规则放在最后的强约束区，防止模型续写时跑偏。

### 例子10：医疗病历分析

先按时间顺序提取病程变化，再归纳异常波动，最后分析诱发因素。

### 方式二：Python 按段落切块

```python
def chunk_by_paragraph(text):
    paragraphs = [p.strip() for p in text.split("\n\n") if p.strip()]
    return paragraphs


sample_text = """
第一段内容。

第二段内容。

第三段内容。
"""

chunks = chunk_by_paragraph(sample_text)
for i, chunk in enumerate(chunks, 1):
    print(f"--- chunk {i} ---")
    print(chunk)
```

### 方式三：Python 固定长度 + 滑动窗口

```python
def sliding_window_chunks(text, chunk_size=1000, overlap=200):
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start += chunk_size - overlap
    return chunks


text = "这里替换成你的长文本" * 200
chunks = sliding_window_chunks(text, chunk_size=500, overlap=100)

for i, chunk in enumerate(chunks[:3], 1):
    print(f"--- chunk {i} ---")
    print(chunk[:200])
```

### 方式四：对每个块做局部摘要

```python
import requests

URL = "http://localhost:11434/api/generate"
MODEL = "gemma3:4b"


def summarize_chunk(chunk_text):
    prompt = f"""
请总结下面这段文本的核心信息。
要求：
1. 输出3点
2. 每点不超过40字
3. 不要编造

文本：
{chunk_text}
"""
    payload = {
        "model": MODEL,
        "prompt": prompt,
        "stream": False,
        "options": {
            "temperature": 0.2,
            "top_p": 0.9
        }
    }
    response = requests.post(URL, json=payload, timeout=120)
    response.raise_for_status()
    return response.json()["response"].strip()
```

### 方式六：上下文排序实验

这段代码用来比较“指令放前面”和“指令放后面”的差异。

```python
import requests

URL = "http://localhost:11434/api/generate"
MODEL = "gemma3:4b"

long_text = "这里替换成长文本材料。"

prompt_a = f"""
请告诉我这份材料最重要的风险点。

材料：
{long_text}
"""

prompt_b = f"""
材料：
{long_text}

请告诉我这份材料最重要的风险点。
"""


def run_prompt(prompt_text):
    payload = {
        "model": MODEL,
        "prompt": prompt_text,
        "stream": False,
        "options": {
            "temperature": 0.2,
            "top_p": 0.9
        }
    }
    response = requests.post(URL, json=payload, timeout=120)
    response.raise_for_status()
    return response.json()["response"].strip()


print("===== 指令在前 =====")
print(run_prompt(prompt_a))
print("\n===== 指令在后 =====")
print(run_prompt(prompt_b))
```

## 课堂实验方案

### 实验目标

让学生亲眼看到：全文直塞、上下文排序、局部证据选择，这三种方式的效果差异。

### 实验准备

准备一份较长材料，在中间埋入一条关键信息。

### 实验步骤

1. 全文直接输入，测试模型能否找出关键信息
2. 仍然全文输入，但把关键指令放最后
3. 只截取相关证据片段，再提问
4. 比较三轮结果

### 观察点

- 哪种方式最稳定
- 哪种方式最慢
- 哪种方式最容易产生幻觉
- 哪种方式最适合真实工作流

## 课堂流程

1. 讨论“把全文直接贴进去”的问题
2. 演示切块
3. 演示排序和摘要压缩
4. 对比不同组织方式的效果

## 教师提醒

### 提醒1

不要迷信超大上下文窗口。窗口大，不等于模型真的会均匀关注所有内容。

### 提醒2

多文档输入时，一定要用清晰标记把文档隔开。

### 提醒3

分块不要机械一刀切，最好考虑语义边界，必要时加滑动窗口。

### 提醒4

最关键的任务要求、输出格式和限制条件，通常放在材料之后更稳。

## 常见误区

- 误以为上下文越长越好
- 误以为材料都给全最安全
- 误以为长文任务只靠模型自己读就够
- 误以为摘要压缩会一定损失所有细节
- 误以为 RAG 只适合程序员，不适合普通知识工作者

## 核心问题

如何把零散的有效提问积累成可复用模板库。

## 这节学什么

学习如何从一次性好 prompt 中抽取可迁移结构，形成个人模板库。

## 为什么这节重要

很多学习者在刚接触提示词工程时，会陷入一个常见问题：今天写出一个好 prompt，明天就找不到了；或者换了一个任务，之前那条好 prompt 完全用不上。

这说明你拥有的是“偶然有效的一次性提问”，而不是“可迁移、可复用、可持续优化的模板资产”。这节课的目标，就是把零散 prompt 变成个人方法库。

## 关键概念

- reusable template
- prompt pattern
- rewrite
- use case

## 通俗解释

### 1. 什么是提示词重写

提示词重写不是把一句话改得更长，而是把一条具体 prompt 中真正有效的结构抽取出来，去掉只对当前场景有效的偶然内容。

例如：

```text
请帮我为英语专业学生设计一节关于命名实体识别的课程，要有例子、误区和练习。
```

这条 prompt 也许很好用，但它只直接适合一个具体场景。重写之后，可以抽出更稳定的结构：

```text
对象：
任务：
输入资料：
输出要求：
例子要求：
限制条件：
```

这样你以后换成“情感分析”“RAG”“提示词评测”时，也能复用同一套骨架。

### 2. 什么是模板库

模板库不是“收集一堆 prompt 原文”，而是把常用任务按类型归档，形成可重复调用的提示结构集合。

一个最基本的模板库，通常至少会包括：

- 摘要模板
- 分类模板
- 写作模板
- 改写模板
- 提取模板
- 表格输出模板
- 教学解释模板
- 计划生成模板

### 3. 为什么一次性好 prompt 不等于模板

一次性好 prompt 往往包含太多具体场景信息，例如：

- 特定主题
- 特定对象
- 特定语气
- 特定材料

这些内容在当下有效，但一旦换场景，prompt 就失去复用性。模板需要保留的是“稳定结构”，不是“场景细节全文”。

### 4. 提示词重写的核心动作

提示词重写通常包括四步：

1. 找出这条 prompt 为什么有效
2. 区分稳定结构和临时内容
3. 把临时内容抽成可替换变量
4. 给这个结构命名并标注适用场景

## 一套可操作的模板抽取框架

你可以用下面这个方法，把一条好 prompt 改造成模板：

### 第一步：保留结构

先看这条 prompt 里有哪些结构性部分：

- 任务
- 对象
- 材料
- 输出格式
- 约束
- 示例要求

### 第二步：替换变量

把具体内容替换成占位符，例如：

- `[主题]`
- `[目标学习者]`
- `[输入材料]`
- `[输出格式]`

### 第三步：命名模板

例如：

- 教学解释模板
- 论文摘要模板
- 评论分类模板
- 项目规划模板

### 第四步：写适用边界

每个模板都要说明：

- 适合什么任务
- 不适合什么任务
- 通常需要哪些输入

## 补充模块：稳定骨架与动态细节

这是提示词重写里最关键、也最容易被忽略的一步。

### 什么是稳定骨架

稳定骨架是这类任务里反复出现、通常不会变的结构部分，例如：

- 指令方式
- 角色定位
- 输出格式
- 限制条件

这些部分定义的是模型“怎么工作”。

### 什么是动态细节

动态细节是每次任务都会变化的部分，例如：

- 当前主题
- 当前输入材料
- 当前对象
- 当前数据内容

这些部分定义的是模型“处理什么”。

### 一个简单判断方法

你可以问自己两个问题：

1. 这部分下次换任务时还大概率保留吗
2. 这部分更像规则，还是更像本次素材

如果答案偏向“规则”，它更可能属于稳定骨架；如果偏向“本次素材”，它更可能属于动态细节。

### 例子

在一个“竞品分析模板”里：

- “请从市场定位、核心功能、商业模式三个维度比较”是稳定骨架
- “下面是 A 公司和 B 公司的介绍材料”是动态细节

## 补充模块：模板命名规范

模板一多，如果没有命名规则，很快就会乱。

### 推荐命名法

建议用“三段式”命名：

```text
功能分类 + 主题域 + 版本状态
```

例如：

- `提取_会议纪要行动项_v1_Active`
- `生成_课程简介_v1_Draft`
- `改写_正式商务邮件_v2_Active`

### 为什么这样命名

因为它同时回答了三个问题：

- 这是干什么的
- 适用于什么场景
- 它现在是不是成熟版本

### 可用状态

- `Draft`：草稿版，仍在测试
- `Active`：稳定可用
- `Retired`：已停用

## 补充模块：如何写模板边界

一个成熟模板，不只要告诉你“怎么用”，还要告诉你“什么时候别乱用”。

### 对人写的边界说明

建议在模板说明里写清楚：

- 适合什么任务
- 不适合什么任务
- 推荐搭配什么模型或参数

例如：

```text
本模板适合提取结构化信息，不适合开放式创意写作。
建议配合低 temperature 使用。
```

### 对模型写的限制条件

在模板正文里，也要写清边界：

- 只能基于输入文本
- 不得补充外部事实
- 如果缺字段就输出 null

这样模板才有真正的护栏。

## 补充模块：Prompt Logbook

模板库不是静态收藏夹，它应该和一个实验记录台配合使用。

### 最低字段

建议至少记录：

- 目标
- 模型
- 参数
- prompt 版本
- 输出结果
- 效果评价
- 下一轮修改点

### 为什么要有 Logbook

因为模板不是写出来就永远正确。你需要知道：

- 哪个版本更稳定
- 哪个版本更适合某类任务
- 哪次修改真的带来了提升

## 补充模块：把成熟框架固化进模板

高质量模板不一定只是“任务+格式”，也可以把成熟分析框架嵌进去。

### 1. SCQA 模板

适合：

- 工作汇报
- 项目汇报
- 方案陈述

结构：

- Situation
- Complication
- Question
- Answer

### 2. TOBY 模板

适合：

- 文献分析
- 报告解构
- 深度阅读

可以让模型依次处理：

- 定位背景
- 解构论证
- 质疑局限
- 产出行动建议

### 3. SOMA 模板

适合：

- 输出评估
- 打分反馈
- 多维度评价

重点是让模型按维度、按量表、按标准评估，而不是泛泛评价。

## 8个可直接上手的模板例子

### 例子1：摘要模板

原始 prompt：

```text
请帮我总结这篇论文，讲清方法、结果和局限。
```

模板化版本：

```text
任务：总结下面材料的核心内容。
对象：面向[目标读者]。
输入资料：[文章/论文/报告正文]
输出要求：按[方法/结果/局限]三个部分输出。
限制条件：只基于输入材料，不补充原文没有的信息。
```

### 例子2：分类模板

```text
任务：对以下内容进行分类。
输入资料：[待分类文本]
输出要求：每条内容只输出一个标签。
标签集合：[标签1, 标签2, 标签3]
限制条件：不要输出解释。
```

### 例子3：写作模板

```text
任务：围绕[主题]完成一篇[文章类型]。
对象：面向[目标读者]。
输出要求：包含[标题/引言/主体/结语]，字数约[字数]。
限制条件：语言风格为[正式/通俗/教学型]，避免[某类表达]。
```

### 例子4：教学解释模板

```text
任务：解释[概念]。
对象：面向[零基础学生/本科生/教师]。
输出要求：包含一句话定义、生活化比喻、最小例子、常见误区。
限制条件：不要默认读者具备[某类背景]。
```

### 例子5：研究辅助模板

```text
任务：分析[研究主题]。
对象：面向[研究计划/论文写作/课题申报]场景。
输入资料：[文献摘要/研究设想/数据描述]
输出要求：按研究问题、解释框架、可能假设、局限性输出。
限制条件：论证必须清楚，不要编造文献。
```

### 例子6：表格输出模板

```text
任务：比较或整理[对象A]与[对象B]。
输入资料：[参数说明/文本资料]
输出要求：Markdown表格，字段为[字段列表]。
限制条件：只保留客观信息，不加主观评价。
```

### 例子7：计划生成模板

```text
任务：设计[课程/项目/活动]方案。
对象：面向[使用场景]。
输入资料：[背景材料]
输出要求：包含目标、步骤、时间安排、风险与应对。
限制条件：控制在[预算/时长/资源]范围内。
```

### 例子8：内容改写模板

```text
任务：把下面文本改写为[目标风格]。
输入资料：[原始文本]
输出要求：保持核心信息不变，改成[正式/通俗/教学/营销]版本。
限制条件：不要新增事实，不要删掉关键数据。
```

## 模板库怎么组织

建议至少用下面三种字段整理：

### 1. 模板名称

例如：

- `教学解释模板`
- `摘要模板`
- `表格对比模板`

### 2. 适用场景

例如：

- 适合课程设计
- 适合论文导读
- 适合产品分析

### 3. 模板正文

也就是可以复制后直接替换变量使用的版本。

如果你想进一步规范，可以再加：

- 输入要求
- 输出示例
- 常见失败点

## 建议的模板库目录结构

如果你在 Obsidian 中整理模板，建议至少按下面方式分：

- 写作类模板
- 提取类模板
- 分类类模板
- 解释类模板
- 规划类模板
- 评测类模板

每张模板卡片建议包含：

- 模板名称
- 适用场景
- 模板正文
- 变量说明
- 使用边界
- 最近一次修改记录

### 方式四：批量整理零散 prompt

```python
import requests

URL = "http://localhost:11434/api/generate"
MODEL = "gemma3:4b"

raw_prompts = [
    "请帮我总结这篇论文，讲清方法、结果和局限。",
    "请把这些用户评论分成正面、负面和中立。",
    "请把下面口语化汇报改成正式邮件。",
    "请为零基础学生解释什么是向量数据库，并给例子。",
    "请帮我设计一个两小时的新员工时间管理培训方案。"
]


def extract_template(raw_prompt):
    prompt = f"""
请把下面这条具体 prompt 重写成可复用模板。

要求：
1. 输出模板名称
2. 输出适用场景
3. 输出模板正文
4. 用占位符替换具体内容
5. 不要输出多余解释

原始 prompt：
{raw_prompt}
"""

    payload = {
        "model": MODEL,
        "prompt": prompt,
        "stream": False,
        "options": {
            "temperature": 0.3,
            "top_p": 0.9
        }
    }
    response = requests.post(URL, json=payload, timeout=120)
    response.raise_for_status()
    return response.json()["response"].strip()


for item in raw_prompts:
    print("=" * 80)
    print("原始 prompt：", item)
    print("-" * 80)
    print(extract_template(item))
    print()
```

### 方式五：把模板库保存成 CSV

```python
import csv
import requests

URL = "http://localhost:11434/api/generate"
MODEL = "gemma3:4b"

raw_prompts = [
    "请帮我总结这篇论文，讲清方法、结果和局限。",
    "请把这些用户评论分成正面、负面和中立。",
    "请把下面口语化汇报改成正式邮件。",
    "请为零基础学生解释什么是向量数据库，并给例子。",
    "请帮我设计一个两小时的新员工时间管理培训方案。"
]


def extract_template(raw_prompt):
    prompt = f"""
请把下面这条具体 prompt 重写成可复用模板。

要求：
1. 输出模板名称
2. 输出适用场景
3. 输出模板正文
4. 用占位符替换具体内容
5. 不要输出多余解释

原始 prompt：
{raw_prompt}
"""

    payload = {
        "model": MODEL,
        "prompt": prompt,
        "stream": False,
        "options": {
            "temperature": 0.3,
            "top_p": 0.9
        }
    }
    response = requests.post(URL, json=payload, timeout=120)
    response.raise_for_status()
    return response.json()["response"].strip()


rows = []
for raw_prompt in raw_prompts:
    template_text = extract_template(raw_prompt)
    rows.append({
        "model": MODEL,
        "raw_prompt": raw_prompt,
        "template_text": template_text
    })

with open("gemma3_4b_prompt_templates.csv", "w", newline="", encoding="utf-8-sig") as f:
    writer = csv.DictWriter(
        f,
        fieldnames=["model", "raw_prompt", "template_text"]
    )
    writer.writeheader()
    writer.writerows(rows)

print("模板库已保存到 gemma3_4b_prompt_templates.csv")
```

## 课堂练习方案

### 练习目标

帮助学生把“零散有效 prompt”整理成“可命名、可归档、可复用”的模板库。

### 练习材料

让每位学生先拿出自己最近写过的 3 到 5 条 prompt。

### 练习步骤

1. 判断这些 prompt 分别属于什么任务类型
2. 找出它们哪些部分是稳定结构
3. 把具体内容改成占位符
4. 给每条模板命名
5. 补写适用场景和边界

### 观察点

- 是否真的抽出了可替换变量
- 模板是否还保留了足够清晰的任务结构
- 模板是否能迁移到相似任务
- 模板名称是否足够明确

## 课堂流程

1. 收集已有提示词
2. 抽取重复出现的结构
3. 合并为模板
4. 给模板命名和适用场景

## 教师提醒

### 提醒1

不要把模板库做成“prompt 收藏夹”。收藏很多原文，不等于拥有模板系统。

### 提醒2

模板要抽象，但不能抽象到失去执行力。变量太多，模板就会变得空洞。

### 提醒3

模板名称不要太泛。与其叫“万能模板”，不如叫“论文摘要模板”或“教学解释模板”。

### 提醒4

模板一定要写边界。否则学生会把“适合摘要”的模板硬拿去做“研究设计”，结果很差。

## 常见误区

- 误以为好 prompt 原文越多，模板库越强
- 误以为所有任务都可以合并成一个万能模板
- 误以为模板只需要正文，不需要适用场景说明
- 误以为模板一旦建好就不用再改
- 误以为模板库是静态仓库，而不是持续迭代系统

## 课后延伸

- 把你目前最常用的 10 条 prompt 重新整理成 5 类模板
- 给每类模板写一个“失败案例”，帮助识别边界
- 结合 [结构化提示词](../../../30-知识卡片/10-概念卡/结构化提示词/) 继续完善个人模板系统

## 作业

提交一个不少于 8 条的个人模板库，并说明每条模板适用的任务边界。

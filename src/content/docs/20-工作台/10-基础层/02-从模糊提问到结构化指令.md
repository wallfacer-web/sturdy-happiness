## 核心问题

如何把“帮我做一下”改写成模型可执行的明确任务。

## 这节学什么

学习把自然语言需求拆成任务、背景、输入材料和输出要求。

## 为什么这节重要

很多人第一次用大模型时，失败并不是因为模型太差，而是因为提问方式太模糊。人和人交流时，可以靠常识、默契和背景补全信息；但模型不会自动知道你真正要的是什么。你不说清楚，它就只能猜。

所以这节课的核心，不是学会“多写一点字”，而是学会把模糊意图转成清晰、可执行、可复查的结构化指令。

## 关键概念

- task
- context
- input
- output schema

## 通俗解释

### 1. 为什么模糊提问容易失败

像“帮我总结一下”“帮我分析一下”“帮我写一下”这样的提示词，看起来是在提需求，实际上几乎没有告诉模型关键边界。模型不知道：

- 你到底要它做什么
- 你面向谁使用结果
- 结果要长什么样
- 什么能说，什么不能说

这时模型只能选择最保守、最通用的输出，于是你得到的往往是“正确的废话”。

### 2. 什么是结构化指令

结构化指令就是把一个任务拆成几个清楚的组成部分，让模型少猜、多执行。最基础的结构通常包括：

- `任务`：你要模型做什么
- `上下文`：这个任务处在什么场景里
- `输入材料`：模型要基于什么内容工作
- `输出要求`：结果要以什么形式交付
- `限制条件`：什么不能做，什么必须遵守

### 3. 为什么结构化指令更有效

因为它减少了模型的猜测空间。你交代得越清楚，模型需要“脑补”的部分越少，输出就越稳定、越贴近你的真实目标。

## 五个核心模块

### 1. 任务

任务就是一句清晰的动作指令。最好使用明确动词，例如：

- 总结
- 提取
- 分类
- 比较
- 翻译
- 设计

不要只说“帮我弄一下”，这不是任务，只是情绪。

### 2. 上下文

上下文回答的是“这个任务发生在什么场景里”。同样一句“解释一下机器学习”，如果面向对象不同，回答应该完全不同：

- 面向高中生
- 面向产品经理
- 面向研究生

上下文决定模型怎么说，而不仅仅是说什么。

### 3. 输入材料

输入材料是模型要处理的原始内容，例如：

- 一段会议纪要
- 一篇论文摘要
- 一份合同
- 一组评论

如果没有输入材料，模型就只能依靠自己已有知识；如果输入材料给得不清楚，模型就容易混淆任务和材料本身。

### 4. 输出要求

输出要求决定结果的可用性。你要明确：

- 输出是列表、表格还是 JSON
- 要几个要点
- 字数多长
- 语气是正式还是通俗

没有输出要求，模型常常会给你一个“能看但不好用”的回答。

### 5. 限制条件

限制条件是最容易被忽略、但很有价值的一部分。它告诉模型：

- 不要编造
- 只能基于我提供的材料
- 不要用术语
- 不要超过 200 字

限制条件的作用不是“卡模型”，而是给模型边界。

## 典型练习

把 5 个日常模糊任务改写成结构化提示词。

## 最小示例

把“帮我总结这篇文章”改成“请基于下文，用 3 点总结文章核心观点，每点不超过 40 字，并指出 1 个局限”。

## 8个可直接上手的例子

### 例子1：摘要

模糊提问：

```text
总结这篇文章。
```

结构化版本：

```text
任务：总结下面文章的核心内容。
上下文：我是课程教师，要给学生做导读。
输入材料：下面的文章正文。
输出要求：用3点总结核心观点，每点不超过40字。
限制条件：只基于原文，不要补充原文没有的信息。
```

### 例子2：分类

模糊提问：

```text
给这些评论分类。
```

结构化版本：

```text
任务：对以下用户评论进行分类。
上下文：用于产品复盘。
输入材料：下面10条评论。
输出要求：每条评论后只输出一个标签，标签只能是“功能问题、价格问题、体验问题、其他”。
限制条件：不要输出解释。
```

### 例子3：写作

模糊提问：

```text
写一篇关于新能源的文章。
```

结构化版本：

```text
任务：写一篇关于新能源优势的科普文章。
上下文：面向普通公众的公众号推文。
输入材料：无。
输出要求：包含标题、引言、3个小标题和结语，约800字。
限制条件：语言通俗，不要使用复杂术语。
```

### 例子4：教学解释

模糊提问：

```text
什么是向量数据库？
```

结构化版本：

```text
任务：解释什么是向量数据库。
上下文：面向完全没有技术背景的大学生。
输入材料：概念“向量数据库”。
输出要求：包括一句话定义、生活化比喻、最小例子、常见误区。
限制条件：不要使用公式，不要默认读者有编程基础。
```

### 例子5：信息提取

模糊提问：

```text
把合同重点找出来。
```

结构化版本：

```text
任务：提取这份租赁合同中的核心信息。
上下文：我要把信息录入合同管理表。
输入材料：下面的合同全文。
输出要求：输出出租方、承租方、租金、付款周期、违约条款。
限制条件：如果原文没有提到某项，就写“原文未提及”。
```

### 例子6：研究辅助

模糊提问：

```text
帮我分析机器学习在教育中的应用。
```

结构化版本：

```text
任务：分析机器学习在教育中的三类主要应用方向。
上下文：我正在准备一个研究计划书。
输入材料：我提供的三篇文献摘要。
输出要求：按“应用方向、典型场景、潜在风险”三列输出表格。
限制条件：只基于输入材料，不要凭空扩展。
```

### 例子7：项目规划

模糊提问：

```text
帮我做个产品发布会策划。
```

结构化版本：

```text
任务：设计一份产品发布会策划案。
上下文：我们是一家做AI产品的公司，准备发布新工具。
输入材料：产品功能简介如下。
输出要求：包含目标、受众、流程、宣传动作、风险与应对。
限制条件：预算控制在10万元以内。
```

### 例子8：表格输出

模糊提问：

```text
比较这两款手机。
```

结构化版本：

```text
任务：比较两款手机的关键参数。
上下文：面向普通消费者的购买建议场景。
输入材料：两款手机的官方参数说明。
输出要求：用 Markdown 表格输出，列出电池、相机、屏幕、价格、主要差异。
限制条件：只写客观参数，不要加入主观评价。
```

## 一套简单可复用的结构化模板

对于初学者，可以先用下面这个骨架：

```text
任务：
上下文：
输入材料：
输出要求：
限制条件：
```

如果还不熟练，就先保证至少写出前三项：

```text
任务 + 输入材料 + 输出要求
```

### 方式四：批量改写模糊提示词

这段代码适合课堂上批量测试。

```python
import requests

URL = "http://localhost:11434/api/generate"
MODEL = "gemma3:4b"

fuzzy_prompts = [
    "帮我总结这篇文章。",
    "给这些评论分类。",
    "写一篇关于新能源的文章。",
    "解释一下什么是向量数据库。",
    "帮我做个产品发布会策划。"
]


def rewrite_prompt(fuzzy_prompt):
    prompt = f"""
请把下面这条模糊提问改写成结构化指令。

要求：
1. 使用“任务、上下文、输入材料、输出要求、限制条件”五个部分
2. 改写后要具体、可执行、可直接复制使用
3. 不要输出额外解释

模糊提问：
{fuzzy_prompt}
"""

    payload = {
        "model": MODEL,
        "prompt": prompt,
        "stream": False,
        "options": {
            "temperature": 0.3,
            "top_p": 0.9
        }
    }
    response = requests.post(URL, json=payload, timeout=120)
    response.raise_for_status()
    return response.json()["response"].strip()


for item in fuzzy_prompts:
    print("=" * 80)
    print("原始提问：", item)
    print("-" * 80)
    print(rewrite_prompt(item))
    print()
```

### 方式五：把改写结果保存成 CSV

```python
import csv
import requests

URL = "http://localhost:11434/api/generate"
MODEL = "gemma3:4b"

fuzzy_prompts = [
    "帮我总结这篇文章。",
    "给这些评论分类。",
    "写一篇关于新能源的文章。",
    "解释一下什么是向量数据库。",
    "帮我做个产品发布会策划。"
]


def rewrite_prompt(fuzzy_prompt):
    prompt = f"""
请把下面这条模糊提问改写成结构化指令。

要求：
1. 使用“任务、上下文、输入材料、输出要求、限制条件”五个部分
2. 改写后要具体、可执行、可直接复制使用
3. 不要输出额外解释

模糊提问：
{fuzzy_prompt}
"""

    payload = {
        "model": MODEL,
        "prompt": prompt,
        "stream": False,
        "options": {
            "temperature": 0.3,
            "top_p": 0.9
        }
    }
    response = requests.post(URL, json=payload, timeout=120)
    response.raise_for_status()
    return response.json()["response"].strip()


rows = []
for fuzzy in fuzzy_prompts:
    structured = rewrite_prompt(fuzzy)
    rows.append({
        "model": MODEL,
        "fuzzy_prompt": fuzzy,
        "structured_prompt": structured
    })

with open("gemma3_4b_structured_prompt_rewrites.csv", "w", newline="", encoding="utf-8-sig") as f:
    writer = csv.DictWriter(
        f,
        fieldnames=["model", "fuzzy_prompt", "structured_prompt"]
    )
    writer.writeheader()
    writer.writerows(rows)

print("结果已保存到 gemma3_4b_structured_prompt_rewrites.csv")
```

## 课堂练习方案

### 练习目标

帮助学生掌握“找缺口 -> 补结构 -> 测效果”的基本方法。

### 练习材料

给学生一条模糊提问：

```text
帮我准备一下明天开会的资料。
```

### 练习步骤

1. 让学生先判断这句话缺了什么
2. 用五个模块补全它
3. 输入模型测试结果
4. 对比改写前后的输出质量
5. 再做第二轮优化

### 观察点

- 模型是否更理解任务
- 输出是否更具体
- 是否更容易直接使用
- 是否减少了空话和跑题

## 课堂流程

1. 展示几个低质量提示词
2. 识别它们缺了什么
3. 现场重写
4. 比较改写前后的输出

## 教师提醒

### 提醒1

不要把 AI 当成有默契的人类同事。你知道的背景，模型未必知道。

### 提醒2

结构化不是为了写得更长，而是为了写得更清楚。

### 提醒3

如果任务和限制条件互相冲突，模型一定会表现不稳定。比如你既要求“深度分析”，又要求“50字以内”，这本身就不自洽。

### 提醒4

处理长文本时，最好把“指令”和“材料”用明显分隔符隔开，避免模型把两者混在一起。

## 常见误区

- 误以为多写几句空话就等于结构化
- 误以为角色设定能代替任务说明
- 误以为不给输出格式也没关系
- 误以为失败说明模型太笨，而不是提示词有歧义
- 误以为所有任务都用同一个提示模板就够了

## 课后延伸

- 用同一个模糊提问，分别写出“教学版、研究版、产品版”三种结构化提示词
- 比较“只写任务”和“写完整五模块”时，输出差异有多大
- 结合 [结构化提示词](../../../30-知识卡片/10-概念卡/结构化提示词/) 继续整理你自己的模板库
